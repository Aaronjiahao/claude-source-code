## 一、系统定位：什么是 DimOS？

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        DimOS: The Agentive Operating System                 │
│                          面向物理空间的智能体操作系统                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│    传统机器人系统                      DimOS                                 │
│    ┌──────────────┐                   ┌─────────────────────────────────┐   │
│    │  ROS/ROS2    │                   │  Agent-Native Architecture      │   │
│    │  中心式节点   │    ────────▶      │  原生智能体架构                  │   │
│    │  复杂配置     │                   │  声明式模块组合                  │   │
│    │  C++为主     │                   │  Python优先                     │   │
│    └──────────────┘                   │  无需ROS                        │   │
│                                       └─────────────────────────────────┘   │
│                                                                             │
│    核心理念："vibecode your robots" - 用自然语言编程机器人                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**DimOS** 是一个现代化的通用机器人操作系统，目标是成为机器人领域的下一代 SDK 标准。它的核心定位是：

- **Agent-Native**: 原生支持 AI Agent 控制，而非传统的手动遥控或预编程行为
- **Hardware-Agnostic**: 支持人形机器人、四足机器人、无人机、机械臂等多种硬件
- **Zero-ROS**: 无需 ROS 环境，纯 Python 开发
- **Compositional**: 通过声明式蓝图(Blueprint)组合模块

### 3.2 认知架构：三层记忆模型

DimOS Agent 拥有**分层记忆系统**，模仿人类认知的感知-工作-长期记忆结构：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        Agent 认知架构：三层记忆模型                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                         长期记忆 (Long-term)                         │  │
│   │  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐              │  │
│   │  │  Spatial     │  │  Temporal    │  │  Semantic    │              │  │
│   │  │  Memory      │  │  Memory      │  │  Memory      │              │  │
│   │  │  空间位置     │  │  时序图谱     │  │  语义知识     │              │  │
│   │  │  (SQLite)    │  │  (Video RAG) │  │  (OSM/VLM)   │              │  │
│   │  └──────────────┘  └──────────────┘  └──────────────┘              │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                    ▲                                        │
│                                    │ 检索/存储                               │
│   ┌────────────────────────────────┴────────────────────────────────────┐  │
│   │                      工作记忆 (Working Memory)                       │  │
│   │  ┌──────────────────────────────────────────────────────────────┐  │  │
│   │  │  Rolling Summary  +  Entity Roster  +  Current Context       │  │  │
│   │  │  (滚动摘要)            (实体名册)         (当前上下文)          │  │  │
│   │  └──────────────────────────────────────────────────────────────┘  │  │
│   └────────────────────────────────┬────────────────────────────────────┘  │
│                                    ▲                                        │
│                                    │ 实时输入                                │
│   ┌────────────────────────────────┴────────────────────────────────────┐  │
│   │                      感知记忆 (Sensory Memory)                       │  │
│   │  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐              │  │
│   │  │  Camera      │  │  LiDAR       │  │  Odometry    │              │  │
│   │  │  (Image)     │  │  (PointCloud)│  │  (Pose)      │              │  │
│   │  └──────────────┘  └──────────────┘  └──────────────┘              │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**关键洞察：**
- **感知记忆**: 原始传感器数据，通过流(Stream)实时传递，不持久化
- **工作记忆**: 当前任务相关的上下文，如"正在寻找红色物体"
- **长期记忆**: 持久化存储，包括空间位置、时序事件、语义知识

### 3.4 System Prompt 工程

DimOS 的 System Prompt 是精心设计的**行为契约**，定义了 Agent 的身份、能力和约束：

```python
# dimos/agents/system_prompt.py

SYSTEM_PROMPT = """
You are Daneel, an AI agent created by Dimensional to control a Unitree Go2 quadruped robot.

# CRITICAL: SAFETY
Prioritize human safety above all else. Respect personal boundaries.
Never take actions that could harm humans, damage property, or damage the robot.

# IDENTITY
You are Daneel. When greeted, briefly introduce yourself as an AI agent
operating autonomously in physical space.

# COMMUNICATION
Users hear you through speakers but cannot see text. Use `speak` to communicate
your actions or responses. Be concise—one or two sentences.

# SKILL COORDINATION

## Navigation Flow
- Use `navigate_with_text` for most navigation.
  It searches tagged locations first, then visible objects, then the semantic map.
- Tag important locations with `tag_location` so you can return to them later.
- During `start_exploration`, avoid calling other skills except `stop_movement`.

## Location Awareness
- `where_am_i` gives your current street/area and nearby landmarks
- `map_query` finds places on the OSM map by description and returns coordinates

# BEHAVIOR

## Be Proactive
Infer reasonable actions from ambiguous requests. If someone says "greet the new arrivals,"
head to the front door. Inform the user of your assumption.
"""
```

**Prompt 设计原则：**
1. **安全第一**: 最顶层强调物理安全
2. **身份明确**: 定义 Agent 的名字和能力边界
3. **工具指导**: 指导如何组合使用技能
4. **行为约束**: 定义主动性和沟通风格

### 3.5 技能发现与注册机制

```
│   │           """                                                      │  │
│   │           ...                                                      │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                    │                                        │
```

## 四、代码框架：模块系统 (Module System)

DimOS 的核心架构是**模块系统**。这是一个基于**响应式流 (Reactive Streams)** 的组件化框架。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        Module 模块系统架构                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────┐      │
```

## 十一、值得学习的设计亮点

### 10.1 声明式编程范式

```python
# DimOS 的声明式蓝图系统让复杂系统搭建变得简单
from dimos.core.blueprints import autoconnect
from dimos.robot.unitree.go2.connection import go2_connection
from dimos.agents.agent import agent
from dimos.perception.experimental.temporal_memory import temporal_memory

# 三行代码搭建一个带视觉记忆的智能机器人系统
blueprint = autoconnect(
    go2_connection(),      # 机器人连接
    agent(),               # LLM Agent
    temporal_memory(),     # 时序记忆
).build().loop()
```

### 10.2 类型驱动的自动连接

```
传统方式需要显式配置每个连接：
    node_a.output_topic = "/camera"
    node_b.input_topic = "/camera"

DimOS 自动连接：
    ModuleA.color_image: Out[Image]  ───┐
                                        ├── 自动匹配 (name, type)
    ModuleB.color_image: In[Image]  ────┘
```

### 10.3 响应式流处理 (RxPY)

```python
# 使用响应式编程处理传感器流
self.connection.lidar_stream().subscribe(self.lidar.publish)

# 背压控制 (Backpressure) 防止内存溢出
observable.pipe(
    backpressure(),       # 自动丢弃旧数据
    ops.filter(lambda x: x.confidence > 0.8),
    ops.throttle_first(1.0),  # 限流
)
```
